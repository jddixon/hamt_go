hamt_go

Go implementation of the HAMT data structure.

A Hash Array Mapped Trie ([bagwell2002][HAMT]) provides fast and memory- efficient access to large amounts of data held in memory. In this particular implementation there are 32- and 64-bit implementations of the code. HAMT associates keys and values. The key is something that produces a 32- or 64-bit unsigned integer, as appropriate. The value is completely arbitrary.

Limitations

  • This code is not thread-safe. That is, using code must provide any necessary locking.

  • HAMT depends upon bit-counting. On modern Intel and AMD processors this is done using a specific machine-language instruction, POPCNT. The current implementation of hamt_go emulates this using the SWAR algorithm, The emulation code is on the order of ten times slower than the machine instruction.

Project Status

Both 32- and 64-bit versions of the code work and are reasonably well-tested. Insert, find, and delete operations, while not yet thoroughly optimized, take on the order of 10 microseconds each on a lightly-loaded server.
